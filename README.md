# 🩺 MediBot — AI Medical Assistant

A domain-specific medical chatbot powered by **Retrieval-Augmented Generation (RAG)** and an **AI Agent** running on **Llama 3.3 70B** via Groq Cloud.

---

## Features

- **RAG pipeline** — retrieves relevant passages from a medical encyclopedia before answering
- **FAISS vector search** — fast semantic similarity search over chunked PDF content
- **AI Agent** — Llama 3.3 70B autonomously decides whether to use the knowledge base or the symptom checker
- **Symptom checker tool** — rule-based triage for common symptom queries
- **Conversation memory** — remembers context across turns within a session
- **Streamlit UI** — dark medical-themed chat interface with sidebar stats

## Setup

### 1. Clone and create virtual environment

```bash
git clone https://github.com/your-username/medibot.git
cd medibot
python -m venv venv

# Activate
venv\Scripts\activate        # Windows
source venv/bin/activate     # Mac / Linux
```

### 2. Install dependencies

```bash
pip install -r requirements.txt
```

### 3. Get a free Groq API key

Sign up at [console.groq.com](https://console.groq.com) — no credit card required.

```bash
# Windows
set GROQ_API_KEY=your_key_here

# Mac / Linux
export GROQ_API_KEY=your_key_here
```

> **Tip — make it permanent on Windows:** open *System Properties → Environment Variables* and add `GROQ_API_KEY` there so you never have to set it again.

### 4. Add your medical PDF

Place your PDF inside the `data/` folder:

```
data/
└── The_GALE_ENCYCLOPEDIA_of_MEDICINE_SECOND.pdf
```

### 5. Build the vector store

Run once to chunk the PDF, embed it, and save the FAISS index:

```bash
python ingest.py
```

### 6. Run the app

```bash
streamlit run app.py
```

Open [http://localhost:8501](http://localhost:8501) in your browser.

## Architecture

```
User Query
    │
    ▼
AI Agent  (Llama 3.3 70B via Groq)
    │
    ├──► 📚 Medical Knowledge Base tool
    │         PDF → chunks → FAISS → top-3 passages → LLM answer
    │
    └──► 🔍 Symptom Checker tool
              keyword match → rule-based triage response
```

## Tech Stack

| Layer | Technology |
|---|---|
| LLM | Llama 3.3 70B (Groq Cloud) |
| Agent framework | LangChain |

| Embeddings | sentence-transformers/all-mpnet-base-v2 |

## Notes

- The vector store (`vectorstore/`) and `data/` folder are excluded from git. Re-run `ingest.py` after cloning on a new machine.
- The GROQ_API_KEY is never committed — always set it as an environment variable.
- First run will download the embedding model (~400 MB) from HuggingFace automatically.
